Git ou le version-ning
Une nouvelle façon de coder

L'intérêt majeur d'utiliser Git est de faciliter le travail sur de gros mais aussi et surtout de pouvoir travailler à plusieurs sans être proche physiquement.
On peut travailler sur des morceaux de codes et les tester sans altérer le code principal qui pourra, lui, évoluer parallèlement.
On dit alors que l'on travaille sur des "branches" que l'on pourra fusionner plus tard avec le code principal (la branche "master").
Git permet, en outre, de revenir "en arrière" dans son code tout en pouvant ensuite revenir à la dernière version. Cela permet entre autre de trouver l'origine de bugs.
Chaque "commit" que l'on effectue est un marquage dans une sorte d'historique qui permet de revenir au code tel qu'il était à ce moment là.
Comme avec la DeLorean de Marty McFly.
La commande "log" par exemple, affichera tous les "commits" qui ont été effectués et l'on pourra choisir dans lequel envoyer la DeLorean.
Nous verrons alors le code tel qu'il était à la date du dit "commit".
"Git possède également l'avantage d'un système de gestion de versions déconnecté. C’est-à-dire qu’il est possible de travailler en mode hors connexion avec son dépôt tout en gardant des fonctionnalités avancées de collaboration lorsqu’on est connecté.
De plus, Git est très performant puisqu'il intègre beaucoup d’outils internes (notamment pour l’utilisation des branches)."
